🔧 什么是 MCP？

Model Context Protocol (MCP) - 模型上下文协议

MCP 是一个开放标准协议，旨在让 AI 应用程序能够安全地访问本地和远程资源。通过 MCP，AI 助手可以与各种工具、服务和数据源进行交互，大大扩展了 AI 的能力边界。

简单来说，MCP 就像是给 AI 装上了"手"和"眼睛"，让它能够真正地操作工具、访问数据，而不仅仅是生成文本。

MCP 的核心概念

🔌

服务器 (Server)

MCP 服务器提供特定的功能和资源，例如访问文件系统、数据库、API 等。每个服务器专注于特定领域的功能。

💬

客户端 (Client)

AI 应用程序作为 MCP 客户端，通过协议与服务器通信，调用服务器提供的工具和资源。

🛠️

工具 (Tools)

服务器通过工具向客户端暴露功能，AI 可以根据需要调用这些工具来完成特定任务。

🌟 Chrome DevTools MCP Server

将 Chrome DevTools 的强大功能带给 AI 助手

Chrome DevTools MCP Server 是一个 MCP 服务器实现，它让 AI 助手能够控制和操作 Chrome 浏览器，执行各种自动化任务。这是本次 Workshop 的核心主题。

DevTools MCP 的核心功能

🌐 页面导航与管理

  • 打开新的浏览器标签页
  • 导航到指定 URL
  • 在历史记录中前进/后退
  • 管理多个标签页
  • 关闭不需要的页面

📸 内容捕获与分析

  • 获取页面的可访问性树（a11y tree）快照
  • 截取整个页面或特定元素的屏幕截图
  • 提取页面文本内容和结构
  • 识别页面上的交互元素

🖱️ 页面交互操作

  • 点击按钮、链接等元素
  • 在输入框中填写文本
  • 选择下拉菜单选项
  • 上传文件
  • 拖放操作
  • 鼠标悬停效果

🔍 开发者工具功能

  • 查看控制台消息（console logs、errors、warnings）
  • 监控网络请求和响应
  • 执行自定义 JavaScript 代码
  • 检查 DOM 结构
  • 分析页面性能

⚙️ 环境模拟

  • 调整浏览器窗口大小（测试响应式设计）
  • 模拟不同的网络条件（3G、4G、离线等）
  • 模拟 CPU 节流（测试低性能设备）
  • 处理浏览器对话框（alert、confirm、prompt）

📊 性能分析

  • 启动和停止性能追踪
  • 记录页面加载性能
  • 分析 Core Web Vitals 指标
  • 识别性能瓶颈和优化建议

🎓 开始使用 Chrome DevTools MCP

想要亲自体验 Chrome DevTools MCP Server 的强大功能？以下是快速开始的步骤：

  1. 安装 Node.js：确保你的系统上安装了 Node.js 18 或更高版本
  2. 安装 Chrome DevTools MCP：通过 npm 或直接克隆 GitHub 仓库
  3. 配置 MCP 客户端：在你喜欢的 AI 应用中添加 MCP 服务器配置
  4. 启动并体验：运行服务器，开始通过自然语言控制浏览器

💡 提示：访问官方文档获取详细的安装和配置指南！
